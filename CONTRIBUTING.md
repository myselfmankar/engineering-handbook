# Contributing to Engineering Handbook

First of all, thank you for being a part of this! This project is a curated knowledge base for software engineering, and your contributions help make it better for everyone.

## Quick Start
To contribute, follow the **[Contribution Guide](https://myselfmankar.github.io/docs/handbook/contribute)** on our official site.

### ✅ Code of Conduct
- Be respectful and constructive in your feedback.
- Ensure all diagrams are clear and high-resolution.
- Link to credible sources (books, academic papers, engineering blogs) when possible.

### 🏗️ Folder Structure
- Add new categories to `/docs`.
- Use `_category_.json` for custom labeling.
- Keep filenames descriptive and hyphenated (e.g., `rate-limiting.md`).

### 📦 Branching Policy
1. **Fork** the repo.
2. **Feature Branch**: Create a branch for your updates.
3. **Draft PR**: Open a PR as soon as you start working.
4. **Final Review**: Once tests (npm build) pass, I will merge your PR!
